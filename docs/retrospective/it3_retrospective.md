In this iteration, we added many new features that made the app more usable. For example, users are able to search for a ride that is close to their start and end location without having to manually search all rides and figuring out which ones are close. If they cannot find a ride, they can create a ride request without having to retype.

We also added a couple features such as a map display of the user's desired start and end locations compared with the ride's start and end locations, basic profiles, and chat functionality.

On the positive side, we made significant progress in this iteration by adding many crucial features to ensure a better user experience. We are close to implementing all of our key features. We also had better communication and had a better idea of who was working on what. Overall, this iteration went much more smoothly than the previous iterations.

Things that can be improved going forward are better communication of work progress, as there was some confusion towards the end of the iteration. Another thing we can improve is establishing a better-defined PR/merging process, as some members used different processes than others.
