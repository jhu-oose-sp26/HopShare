We made steady progress in this iteration, getting a far better application that fully implements all CRUD operations and is clean and usable. We don't have a landing page or logon yet, but this is what the user will see once they get past the start page and log into the app. Most if not all of our goals for this iteration were achieved and there are only minor changes that can really be made in Iteration 3 to what we completed in this iteration.

With that being said, one of the things that can be improved is the simple filtering system we have currently, it can be improved to cover radius of start and end location more accurately in the future, but for our current purposes, this will be fine and we will most likely move onto other features in Iteration 3. 

The challenges that we faced were mostly merge conflicts and sometimes time management, as many of us were busy during the time period and communication over Slack is sometimes limited. Looking forwards we will try to have a more structured plan and create features more efficiently. This may include using more ``@channel`` pings and meeting up in person outside of class times. All in all, Iteration 2 was far more organized compared to when we were first starting and just getting to know our team, but there are still some kinks to work out.
