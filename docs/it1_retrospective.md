During this iteration, the team successfully implemented both the frontend and backend components of the application. This represents meaningful progress toward our overall project goals. However, we were not able to complete the integration of these components before the deadline.

What Went Well
- The frontend and backend were both implemented with core functionality.
- We were able to integrate the two components (although 1 day late).
- The integration effort helped clarify how the system components interact, which should reduce conflicts in future merges.

Challenges
- Many tasks were completed later than planned, with no communication on when they would be completed, which limited the time available for integration.
- In a few cases, delayed code uploads prevented other team members from making progress, creating bottlenecks.
- Integration was compressed into the final day, resulting in a rushed merge process.
- There was insufficient alignment on data structures and the variables stored in the database, which led to avoidable adjustments during integration.

Key Takeaways
- Timely completion and uploading of work is critical, especially when tasks are interdependent.
- When deadlines cannot be met, earlier communication would help the team adjust and redistribute work if necessary.
- Clear agreement on data models and database schema upfront would reduce integration friction.
- Scheduling dedicated integration checkpoints earlier in the iteration could mitigate last-minute scrambling.

Action Items for Next Iteration
- Set and commit to intermediate internal deadlines, particularly for dependent tasks.
- Communicate proactively if timelines shift.
- Finalize and document the database schema and expected data structures before implementation begins.
- Allocate time earlier in the iteration for integration and testing.
- We will create documentation for the backend APIs to prevent future confusion on how the database works.

Overall, while we faced coordination and communication challenges, the successful integration of frontend and backend (despite the lateness) provides a stronger foundation for the next iteration. With improved deadline management and clearer alignment on shared data structures, we expect smoother collaboration moving forward.
